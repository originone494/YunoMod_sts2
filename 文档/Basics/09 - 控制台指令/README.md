## 使用方式

塔2内置控制台。在加载mod状态下按`~`开启。

* `tab`键可以根据你当前的文字给出补全候补，按`↑` `↓`方向键选取然后`enter`选择。

* `↑`可以快速选取上一次输入的指令。按`↑` `↓`方向键选取。

* 当选取最新的指令时，`↓`可以直接删除当前内容。

## 全部指令

以下展示所有指令。

* `<X>`表示必填参数，`[X]`表示可选参数。

* `目标索引`从0（你扮演的玩家）开始，单机模式下1往后为敌人（按生成顺序）。联机模式下，1往后先是其他玩家。

### 战斗相关

| 名称 | 指令形式 | 说明 |
|:---:|:---|:---|
| **伤害** | `damage <数值> [目标索引]` | 对所有敌人造成伤害。指定索引可指定单一目标。 |
| **格挡** | `block <数值> [目标索引]` | 为玩家添加格挡。指定索引可为指定生物添加。 |
| **斩杀** | `kill [目标索引\|all]` | 杀死指定目标。`all`杀死全部敌人，默认第一个敌人。 |
| **死亡** | `die` | 直接输。 |
| **胜利** | `win` | 直接赢。 |
| **治疗** | `heal <数值> [索引]` | 恢复玩家生命值。指定索引可治疗其他生物。 |
| **无敌** | `godmode` | 切换无敌模式，给予强力buff。再次输入关闭。 |
| **能力** | `power <ID> <层数> <目标索引>` | 对指定目标施加能力。 |
| **能量** | `energy <数值>` | 为玩家增加能量。可为正数或负数。 |
| **金币** | `gold <数值>` | 为玩家增加金币数量。可为正数或负数。 |
| **辉星** | `stars <数值>` | 为玩家添加指定数量的辉星。 |
| **遗物** | `relic [add\|remove] <遗物ID>` | 为玩家添加或移除遗物。默认为添加。 |
| **药水** | `potion <ID>` | 添加一瓶指定ID的药水。 |

### 卡牌相关

| 名称 | 指令形式 | 说明 |
|:---:|:---|:---|
| **获得卡牌** | `card <卡牌ID> [牌库名]` | 生成卡牌到指定牌堆：`hand`（手牌，默认）、`draw`（抽牌堆）、`discard`（弃牌堆）、`exhaust`（消耗堆）、`master_deck`（主牌组）。 |
| **移除卡牌** | `remove_card <ID> [牌库名]` | 从指定牌库移除卡牌。 |
| **升级卡牌** | `upgrade <手牌位置>` | 升级手牌中指定位置的卡牌（0=最左侧）。 |
| **附加附魔** | `enchant <ID> [层数] [手牌位置]` | 对手牌中的卡牌附加指定附魔。层数默认为1。默认最左侧手牌。 |
| **附加诅咒** | `afflict <ID> [层数] [手牌位置]` | 对手牌中的卡牌附加指定诅咒。层数默认为1。默认最左侧手牌。 |
| **抽牌** | `draw <数量>` | 抽指定数量的牌。 |

## 地图相关

| 名称 | 指令形式 | 说明 |
|:---:|:---|:---|
| **跳转章节** | `act <幕数\|名称>` | 跳转到指定章节。可以用整数或者章节的ID。 |
| **跳转房间** | `room <ID>` | 跳转到指定ID的房间节点。（例如`BOSS`、`SHOP`等，看tab补全） |
| **跳转事件** | `event <ID>` | 跳转到指定ID的事件节点。 |
| **跳转战斗** | `fight <ID>` | 跳转到指定ID的怪物遭遇战。 |
| **地图传送** | `travel` | 切换地图传送模式，可点击任意房间直接跳转。 |
| **先古之民** | `ancient <ID> [遗物ID]` | 跳转到指定ID的先古之民。<br>可添加额外的参数使得必定出现指定ID的遗物选项。 |

## 其他

| 名称 | 指令形式 | 说明 |
|:---:|:---|:---|
| **帮助** | `help [命令名]` | 列出所有可用命令，或查看指定命令的详细帮助和用法示例，例如`help card`。 |
| **打开路径** | `open <logs\|saves\|root\|build-logs\|loc-override>` | 在文件管理器中打开常用目录：日志、存档、游戏根目录、构建日志、本地化覆盖。 |
| **解锁内容** | `unlock <类型>` | 标记指定类型内容为已发现：`cards`、`potions`、`relics`、`monsters`、`events`、`epochs`、`ascensions`。`all` 解锁全部。 |
| **成就** | `achievement <操作> [ID]` | 解锁（`unlock`）或撤销（`revoke`）成就。不提供 ID 则操作全部成就。 |
| **排行榜** | `leaderboard [选项] [名称] <分数> [数量]` | 向排行榜提交分数。选项包括：`upload` 上传，`random` 生成随机分数测试。 |
| **获取日志** | `getlogs [test-feedback] <名称>` | 收集日志并打包为zip，打开所在目录。`test-feedback`打开后仅打包关键文件（适合给游戏开发团队而不是mod开发者） |
| **打印ID** | `dump` | 将所有model的ID输出到控制台和日志文件。 |
| **日志级别** | `log [类型] <级别>` | 设置日志输出级别。级别：`verydebug`、`debug`、`info`、`warn`、`error`。 |
| **缺失美术** | `art <类型>` | 列出指定类型中缺少美术资源的内容。类型：`card`、`relic`、`potion`、`enchant` 等。 |
| **加速模式** | `instant` | 开启加速模式，跳过所有动画延迟。 |
| **图鉴** | `bestiary` | 打开怪物图鉴界面。 |
| **sentry测试** | `sentry <test\|message\|exception\|crash\|status> [文本]` | 测试Sentry错误报告功能。 |
| **预告模式** | `trailer` | 切换预告模式，启用后可用数字键0-9切换UI显示/隐藏，用于录制预告片和截图。 |
| **云存档** | `cloud delete` | 删除Steam云上的所有存档文件。仅限Steam平台。 |
| **多人模式** | `multiplayer` | 打开多人游戏菜单。加参数`test`打开测试场景。 |
